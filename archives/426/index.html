babun使用docker时的一个问题

2016-08-31

babun 是一个window下的命令行工具集，个人感觉用起来是最舒服的，配好oh-my-zsh非常好用。

最近一直在学习一些docker相关的东西，当在babun中使用命令

docker exec -ti web bash

试图进入容器的时候，返回以下错误：

the input device is not a TTY. If you are using mintty, try prefixing the command with ‘winpty’

mintty看来不行了，需要使用winpty，一个简单解决方案，使用babun-docker就可以了，命令如下：

rm -rf ~/.winpty ~/.babun-docker

curl -s https://raw.githubusercontent.com/tiangolo/babun-docker/master/setup.sh | source /dev/stdin

babun-docker-update

再次使用进入容器：

docker exec -ti web bash

the input device is not a TTY. If you are using mintty, try prefixing the command with ‘winpty’
– babun-docker: Using winpty

好的，已经自动切换成winpty进入容器了（开心），可以使用babun了。

PS： 需要注意的事，执行上面脚本的时候，会从github-cloud.s3.amazonaws.com下载winpty的压缩包，需要翻墙才可以。由于我使用的是shadowsock，所以只需要更改下.babunrc中代理即可：

export http_proxy=http://127.0.0.1:1080
